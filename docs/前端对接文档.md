# 插了个花 · 前端对接文档

面向小程序 / Web 前端开发（可直接喂给 AI 辅助对接）。Swagger 在线调试：`/api/docs`。

---

## 1. 基础约定

- **Base URL**：全局前缀 `/api`，如 `https://flower-api-dev.zhilingtech.com/api`
- **编码**：请求/响应均 `application/json; charset=utf-8`
- **统一响应包裹**：所有接口都返回 `{ code, data, msg }`

```jsonc
// 成功：HTTP 200
{ "code": 0, "data": { /* 业务数据 */ }, "msg": "ok" }

// 失败：HTTP 为真实状态码（400/401/403/404/429/500）
{ "code": 401, "data": null, "msg": "令牌无效或已过期" }
```

> **前端封装建议**：拦截器里判断 `code === 0` 取 `data`，否则按 HTTP 状态 + `msg` 走错误处理。下文各接口的「响应」只写 `data` 部分。
> **toast 类错误（如 429 额度）直接把 `msg` 展示给用户即可**，文案是后端按产品要求写死的。

### 错误码

| HTTP | 含义 | 典型场景 |
|---|---|---|
| 400 | 参数错误 / 校验失败 / 内容审核不通过 | 字段缺失、标题超 16 字、昵称/标题违规、缺少访客标识 |
| 401 | 未认证 | 缺 token 或 token 失效 |
| 403 | 无权限 | 操作非本人资源 |
| 404 | 不存在 | 资源 id 无效（含被级联撤回后） |
| 429 | 请求过频 / 额度用尽 | AI 接口限流（每用户 60s ≤ 10 次）、**AI 每日生成额度（5 次/作品/日）**、短信发码限流 |
| 500 | 服务器错误 | 后端异常 |

### 时间与「自然日」

- 所有 `createdAt` 等为 **ISO 8601 UTC 字符串**，展示格式化（如 `2026.07.16`）由前端处理。
- 「每自然日」类规则（浏览去重、AI 额度）后端按**东八区（Asia/Shanghai）**计算，次日 0 点重置，前端无需关心。

## 2. 鉴权

除标 🔓 外所有接口都要登录，请求头带：

```
Authorization: Bearer <accessToken>
```

### 登录流程（小程序）

```
wx.login() 拿 code
  → POST /api/auth/wechat/login { code }
  → 得 { accessToken, refreshToken, user }
  → 本地存两个 token；之后每个请求带 accessToken
```

### 登录流程（Web · 手机验证码）

```
POST /api/auth/sms/send  { "phone": "138****0000" }   → { "sent": true }
POST /api/auth/sms/login { "phone": "...", "code": "6位验证码", "nickname": "可选" }
  → 同微信登录的响应结构
```

- 发码限流：同 IP 5 次/分钟，同手机号 60 秒冷却；触发按 `msg` 提示。
- **同手机号账号互通**：Web 用手机号登录会并入已绑定该手机号的微信账号。

### 令牌说明

| 令牌 | 用途 | 过期后 |
|---|---|---|
| `accessToken` | 每次请求带 | 请求返回 401 |
| `refreshToken` | 换新 accessToken | 需重新登录 |

- 收到 **401** → 用 `refreshToken` 调 `POST /api/auth/refresh` 换新的一对令牌，重试原请求；refresh 也失效则重新登录。
- refresh 是**一次性轮换**：每次刷新会返回新的 refreshToken，旧的立即失效，务必覆盖保存。

## 3. 接口清单

### 3.1 认证

#### `POST /auth/wechat/login` 🔓
微信登录。传 `phoneCode` 则同步绑定手机号。

```jsonc
// 请求
{ "code": "wx-login-code", "phoneCode": "可选", "nickname": "可选(≤12字)", "avatarUrl": "可选,OSS地址" }
// 响应 data
{ "accessToken": "...", "refreshToken": "...", "user": { /* User */ } }
```

#### `POST /auth/sms/send` 🔓
```jsonc
{ "phone": "13800138000" }   // 请求，大陆手机号
{ "sent": true }             // 响应 data；限流/冷却时非 200，按 msg 提示
```

#### `POST /auth/sms/login` 🔓
```jsonc
{ "phone": "13800138000", "code": "123456", "nickname": "可选(≤12字)" }   // 请求
// 响应 data 同微信登录：{ accessToken, refreshToken, user }
```

#### `POST /auth/refresh` 🔓
```jsonc
{ "refreshToken": "..." }          // 请求
{ "accessToken": "...", "refreshToken": "..." }  // 响应 data
```

#### `POST /auth/logout` 🔓
```jsonc
{ "refreshToken": "..." }   // 请求
{ "success": true }         // 响应 data
```

### 3.3 个人中心（我的作品 / 我的发布 / 我的点赞）

三个列表均为**分页接口**，入参 `page`（默认 1）、`size`（1–50，默认 20），响应 `data` 统一为：

```jsonc
{ "items": [ /* 见各列表 */ ], "total": 42, "page": 1, "size": 20 }
```

**封面统一规则（重要）**：三个列表每项都带 `coverUrl`（原图）与 `thumbUrl`（缩略图）—— `coverUrl` **有关联 AI 图则为最新一张 AI 图，否则为原始创作台预览图**；`thumbUrl` 是 480px webp 小图（可能缺省）。**列表卡片统一贴 `thumbUrl ?? coverUrl`**，不要自己拼接逻辑；详情/下载等高清场景用 `coverUrl`。`materialCount` 为花材数量（卡片「N 花材」直接用）。

#### `GET /users/me/works?page=&size=` — 我的作品

```jsonc
// items[i]（MyWorkItem）
{
  "id": "ckwork…", "userId": "…", "title": "我的插花 07.17",
  "theme": "night", "vaseId": "mat-vase-ink",
  "arrangement": { /* 插花快照，继续编辑用 */ },
  "thumbnailUrl": "创作台预览图",
  "coverUrl": "封面原图（最新AI图 ?? 预览图）",
  "thumbUrl": "封面缩略图（480px webp；缺省则回退 coverUrl）",
  "materialCount": 3,
  "published": true,              // 是否已发布到广场（见下方删除规则）
  "dateKey": "2026-07-17",
  "createdAt": "…", "updatedAt": "…"
}
```

**删除作品**：`DELETE /works/:id` → `{ ok: true }`。
- 删除前先看该项 `published`：为 `true` 时确认弹窗文案用「**该作品已发布，删除后也会同步撤回广场发布**」；确认后调删除接口，后端会**级联撤回关联的广场帖**（其点赞一并清理），广场 feed 与该帖详情随即 404。
- 「继续编辑」：用 `arrangement` 载入创作台。

#### `GET /users/me/posts?page=&size=` — 我的发布

```jsonc
// items[i]（MyPostItem）
{
  "id": "ckpost…", "userId": "…", "authorName": "作者昵称快照",
  "title": "今日花事", "theme": "night",
  "arrangement": { /* 发布时快照，继续编辑用 */ },
  "thumbnailUrl": "发布时展示图快照",
  "coverUrl": "封面原图（关联作品最新AI图 ?? 发布快照）",
  "thumbUrl": "封面缩略图（480px webp；缺省则回退 coverUrl）",
  "materialCount": 2,
  "likeCount": 168, "viewCount": 876,
  "workId": "来源作品id（可能为 null）",
  "auditStatus": "approved",
  "createdAt": "发布时间"
}
```

**撤回发布**：`DELETE /plaza/:id` → `{ ok: true }`；非本人 403 / 不存在 404。撤回后从「我的发布」与广场 feed 同时消失，**不影响本地保存的作品**。

#### `GET /users/me/likes?page=&size=` — 我的点赞

```jsonc
// items[i]（MyLikeItem）
{
  "id": "ckpost…", "userId": "作者id", "authorName": "作者昵称",
  "title": "松间一枝", "theme": "night", "arrangement": { /* … */ },
  "thumbnailUrl": "…", "coverUrl": "封面原图（规则同上）",
  "thumbUrl": "封面缩略图（规则同上）",
  "materialCount": 4,
  "likeCount": 53, "viewCount": 876,
  "workId": null,
  "liked": true,                  // 恒 true（该列表即点赞列表）
  "likedAt": "点赞时间",
  "createdAt": "帖子发布时间"
}
```

**取消点赞**：调 `POST /plaza/:id/like`（见 3.7，toggle），返回 `liked: false` 后把该项从列表移除。
**打开预览**：点卡片进作品预览弹窗时，记得调浏览上报（见 3.7 `POST /plaza/:id/view`）。

### 3.5 素材（内置目录）

#### `GET /materials/catalog` 🔓
内置素材目录。**素材是图片渲染**，直接贴 `imageUrl`。

```jsonc
// 响应 data
{
  "categories": [ { "id": "flower", "label": "花朵" }, { "id":"greenery","label":"枝叶" },
                  { "id":"line","label":"线条" }, { "id":"vase","label":"花器" } ],
  "materials": [
    {
      "id": "mat-rose", "name": "玫瑰", "category": "flower",
      "imageUrl": ".../default-materials/flower/mat-rose.png",   // 目录/选择器缩略图（= 首个样式图）
      "styles": [                                                 // 花/枝/线：2~3 款姿态（数量随素材不等）
        { "styleOption": "rose-full", "name": "玫瑰盛放", "imageUrl": ".../flower/mat-rose/rose-full.png" }
        // …styles 已按推荐顺序排好（首款即默认展示款）
      ]
    },
    {
      "id": "mat-vase-ink", "name": "墨色瓶", "category": "vase",
      "imageUrl": ".../default-materials/vase/mat-vase-ink.png",
      "styles": null                                              // 花器：单样式
    }
  ]
}
```

**渲染规则（重要）**
```
material.styles?.length
  ? 选择器展示 styles 全部姿态（2~3 款，按数组长度渲染，勿写死 6 款），每款贴自己的 styles[i].imageUrl；
    用户选中的 styleOption 记到作品项；默认选中 styles[0]
  : 单样式，直接贴 material.imageUrl
```
- 所有素材图都是**透明底 PNG**（真花实拍抠图），姿态图已把长度烘进去，前端只需做 size/旋转/镜像/透明度等几何变换。

### 3.7 作品 + 创作日历

#### `GET /works/calendar?month=YYYY-MM`
响应 `data`：`{ "2026-07-09": 3, ... }`（按天的作品数，供日历角标）

#### `GET /works?dateKey=YYYY-MM-DD`
响应 `data`：`Work[]`（某天作品，倒序；个人中心列表请用 3.3 的 `/users/me/works`）

#### `GET /works/:id`
响应 `data`：`Work`；403 / 404

#### `POST /works` — 保存作品
```jsonc
{ "title":"我的插花 07.17", "theme":"night", "vaseId":"mat-vase-ink",
  "arrangement": { /* 插花快照，后端不透明 */ },
  "thumbnail":"创作台预览图（dataURL 或 已直传的OSS URL）", "dateKey":"2026-07-09" }
// 响应 data：Work
```
- 保存后自动进入「我的作品」（`/users/me/works`）与创作日历（`dateKey` 即聚合键，**按本地时区生成**）。

#### `PATCH /works/:id`
上述字段任意子集；响应 `data`：`Work`；403 / 404

#### `DELETE /works/:id`
响应 `data`：`{ ok: true }`；403 / 404。**若作品已发布，关联广场帖会被同步撤回（级联删除）**，删除确认文案见 3.3。

#### `GET /works/:id/images` — 下载弹窗图片列表

```jsonc
// 响应 data
{
  "images": [
    { "type": "ai",      "taskId": "cktask…", "url": "最新AI图", "createdAt": "…" },
    { "type": "ai",      "taskId": "cktask…", "url": "更早AI图", "createdAt": "…" },
    { "type": "preview", "url": "当前创作台预览图", "createdAt": "…" }
  ]
}
```

**展示规则（后端已排好序，前端直接用）**：
- `images` 顺序 = 最新 AI 图 → 更早 AI 图 → 创作台预览图垫底。默认选中 `images[0]`。
- **没有 AI 图时数组只有 1 个 `preview` 项** → 只显示大图，**不渲染缩略图区**。
- 作品没有预览图且无 AI 图时返回空数组。

### 3.8 分享广场

#### `GET /plaza?page=&size=&sort=` 🔓
分页 feed（仅 approved）。**公开，无需登录**；登录（带 token）时每条带真实 `liked`，未登录 `liked` 恒 `false`。`page` 从 1 开始；`size` 1–50。

**排序 `sort`**（默认 `latest`，非法值 400）：

| sort | 规则 |
|---|---|
| `latest`（最新） | 发布时间倒序 → 点赞数倒序 |
| `mostLiked`（最赞） | 点赞数倒序 → 发布时间倒序 |
| `hottest`（最热） | 浏览量倒序 → 点赞数倒序 → 发布时间倒序 |

```jsonc
// 响应 data（列表项不含 arrangement——快照单条可达数百 KB；「点开继续编辑」调 GET /plaza/:id 取）
{
  "items": [
    {
      "id": "ckpost…", "userId": "…", "authorName": "作者昵称快照",
      "title": "松间一枝", "theme": "night",
      "thumbnailUrl": "展示图（原图）", "thumbUrl": "展示图缩略图（480px webp；缺省回退 thumbnailUrl）",
      "likeCount": 53, "viewCount": 876,
      "workId": "来源作品id或null", "liked": false,
      "auditStatus": "approved", "createdAt": "…"
    }
  ],
  "total": 128, "page": 1, "size": 8
}
```

#### `GET /plaza/:id`
响应 `data`：`PlazaPost`（含 `liked`、`viewCount`）；404

#### `POST /plaza` — 发布到广场

```jsonc
// 方式一（推荐）：发布已保存的作品
{ "workId": "ckwork…", "title": "今日花事" }
// 方式二：直接给作品信息（无来源作品）
{ "title": "…", "theme": "night", "arrangement": {…}, "thumbnail": "OSS URL" }
// 响应 data：PlazaPost
```

- **`title` 必填，≤ 16 字**，过文本审核（违规 400）。前端默认自动填充「今日花事」，用户可改。
- **展示图规则（后端处理，前端不传）**：方式一发布时，后端自动取「**该作品最新一张 AI 图**」为展示图，没有 AI 图则用创作台预览图，存为帖子的 `thumbnailUrl`。
- 发布后该项即出现在「我的发布」（3.3）。

#### `POST /plaza/:id/like` — 点赞 / 取消点赞
**同一个接口**（幂等 toggle）。
```jsonc
// 响应 data：liked = 本次操作后的状态；计数即时增减
{ "likeCount": 13, "liked": true }
```

#### `POST /plaza/:id/view` 🔓 — 浏览量上报

```jsonc
// 登录用户：空 body 即可
{}
// 未登录用户：必须带匿名 ID（设备 ID / localStorage 持久化的随机 ID）
{ "anonId": "web-7f3c9a…" }
// 响应 data
{ "counted": true, "viewCount": 877 }   // counted=false 表示当日已计过，未重复计数
```

**调用时机（严格遵守，否则数据失真）**：
- **用户点击广场卡片进入作品预览弹窗时**，调 1 次。
- 从「我的点赞」打开广场作品预览，也调 1 次。
- 列表曝光、下拉刷新、切换排序、弹窗内重复操作：**不要调**。
- 去重由后端完成（同一用户/匿名 ID 对同一帖子每自然日最多计 1 次），前端无需自己防重；匿名 ID 需在 localStorage 持久化，未登录且不传 `anonId` 返回 400。

#### `DELETE /plaza/:id` — 撤回发布
作者本人；响应 `data`：`{ ok: true }`；403 / 404。详见 3.3「我的发布」。

### 3.9 图片上传

**推荐直传流程**（后端不扛图片流量）：
```
POST /upload/signature { scene } → 得直传签名
  → 把文件直传 OSS（小程序 wx.uploadFile / Web 表单 POST）
  → 拿 fileUrl，把 URL 传给业务接口（如 works.thumbnail / plaza.thumbnail / cutout.sourceImageUrl）
```

#### `POST /upload/signature`
```jsonc
{ "scene": "work-thumbnail", "ext": "png" }   // scene: cutout-source|export-reference|work-thumbnail|general
// 响应 data：OssUploadSignature（host/key/policy/signature/ossAccessKeyId/expire/maxSize/fileUrl）
```
直传表单字段（POST 到 `host`）：
```js
{ key, OSSAccessKeyId, policy, signature, success_action_status: '200' }
// 成功后用签名响应里的 fileUrl
```

#### `POST /upload`（服务端代传，兜底小图）
`multipart/form-data`（字段 `file`）或 JSON `{ dataUrl }`；≤8MB，png/jpeg/webp。响应 `data`：`{ url }`

### 3.10 AI 能力（异步任务 + 每日额度）

**提交 → 轮询**模式。POST 立即返 `taskId`，再轮询 GET 直到 `status` 终态。限流：每用户 60s ≤ 10 次提交（429）。

`status`：`pending`（入队）→ `running`（处理中）→ `succeeded` / `failed`（终态）。轮询到 `succeeded` 取结果 URL，`failed` 看 `error`（**失败不占每日额度**）。

#### `POST /image2` — 由插花生成摄影风格图

```jsonc
{
  "prompt": "…",
  "referenceImageUrl": "参考图OSS URL(推荐，与 referenceImage 二选一)",
  "size": "1024x1536",
  "workId": "当前作品id（已保存的作品必传；未保存草稿不传）"
}
// 响应 data
{ "taskId": "cktask…", "status": "pending", "progress": 5, "remaining": 4 }
```

**每日额度（重点）**：**每用户对同一作品每自然日（东八区）最多成功生成 5 张**。
- `workId`：已保存的作品**必传**（额度、下载列表、发布展示图、列表封面都按它关联）；创作台尚未保存的草稿不传，草稿共享一个「每用户每日 5 次」的额度桶。
- 提交时若额度已用完 → **HTTP 429**，`msg` 固定为 `今日 AI 生成额度已用完（5/5），明天再来试试吧`，**直接 toast msg**。
- `remaining` = 今日剩余可发起次数（含在途占用）。
- 前端 toast 映射：
  - 点击「AI 生成」立即 toast：`已开始生成，本次将消耗 1 次 AI 生成额度`（前端自理，无需接口数据）。
  - 轮询到 `succeeded` 后 toast：`生成成功，今日还剩 X 次`，X 取轮询响应的 `remaining`。
  - 429 时 toast：上面那句固定 msg。
- **生成失败不计次数**（后端按任务终态统计）。

#### `GET /image2/:taskId`
```jsonc
{ "status": "succeeded", "progress": 100, "imageUrl": "结果OSS URL（原图 PNG）", "thumbUrl": "缩略图OSS URL（480px webp）", "error": null, "remaining": 4 }
```
> 列表/预览贴 `thumbUrl`（几十 KB），保存/下载高清图用 `imageUrl`。`thumbUrl` 生成失败时缺省，回退 `imageUrl`。

#### `POST /cutout-flower` — 上传照片抠图成透明底素材
只传 类型 / 花材名 / 照片链接，prompt 等由后端内置处理。**不受每日 5 次额度限制**（仍受 60s 10 次限流）。
```jsonc
{ "category":"flower", "name":"院子里的月季", "sourceImageUrl":"照片OSS URL" }
// 响应 data：{ taskId, status, progress }
```
#### `GET /cutout-flower/:taskId`
```jsonc
{ "status":"succeeded", "progress":100, "imageUrl":"透明底图OSS URL", "thumbUrl":"缩略图OSS URL（480px webp，保留透明底）", "image":"同 imageUrl（兼容旧字段）", "error": null }
```
> 抠图成功后，拿 `imageUrl` 调 `POST /materials/custom` 入库成花材；花材选择器等小图场景可用 `thumbUrl`。

## 5. 前端要点清单

- [ ] 请求拦截器：带 `Authorization`，解包 `{code,data,msg}`，401 自动 refresh 重试。
- [ ] refresh 轮换：每次刷新覆盖保存新的 refreshToken。
- [ ] 素材目录：`styles?.length` 决定单/多样式渲染。
- [ ] 图片一律走 `/upload/signature` 直传 OSS，业务接口只收 URL。
- [ ] **AI 生成**：`POST /image2` 带 `workId`（未保存草稿不带）；429 直接 toast `msg`；成功 toast 用响应 `remaining`。
- [ ] **下载弹窗**：`GET /works/:id/images`，按返回顺序渲染；仅 1 项时不显示缩略图区。
- [ ] **发布**：标题必填 ≤16 字（默认填「今日花事」）；展示图由后端定（最新 AI 图 ?? 预览图）。
- [ ] **广场**：feed 用 `sort` 切换三种排序；点赞用同一个 like 接口 toggle，按返回 `liked` 更新 UI。
- [ ] **浏览上报**：只在「打开作品预览弹窗」时调 `POST /plaza/:id/view`；未登录带持久化 `anonId`；列表曝光不调用。
- [ ] **个人中心三列表**：封面统一用 `coverUrl`；删作品前看 `published` 决定确认文案；撤发布 `DELETE /plaza/:id`；取消赞走 like toggle。
- [ ] 自定义素材单样式，隐藏姿态面板。
- [ ] 昵称/素材名/广场标题可能因内容审核返回 400，给用户提示。

> ⚠️ **图片内容审核暂未启用**（仅文本审核）；上线合规前需补 `imgSecCheck`。
